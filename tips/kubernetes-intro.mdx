# kubernetesの勉強をはじめた

2月17日、会社のfrontend勉強会でAWS AppSync + GraphQLのハンズオンをした結果、GraphQLの向こう側をやりたくなった。

### まずこの記事を読んだ
https://zenn.dev/esaka/articles/2d117655af1f03cf2444

物理サーバーでそのまま動かしていた時代から、仮想化→クラウド→コンテナ→オーケストレーションが生まれた経緯が分かりやすかった。
記事中でサイドカーパターンと呼ばれている、サービス機能のアプリケーションにプロキシやロギングなどの機能を簡単に付けられるという特徴が、AWS EC2でインスタンスを立てて自分でいろいろ入れて設定をしていた時代で止まっていた自分としては強い良さを感じた。補助的なものをコンテナという単位で分けて構築できると、作りたい機能の実装に集中できる環境が作りやすそう。

### Minikubeのチュートリアルをする
https://kubernetes.io/ja/docs/tutorials/hello-minikube/

Minikubeは本来クラウドや複数サーバーで動かすkubernetesをローカルで動かせる様にしたもの(の認識)。
まだ最初しかできていない。環境構築と起動をしていて新しいPCがほしくなった。

何か都合の良いAPIサーバーとかを書いて実行させる。もしくはこのブログをkubernetesに乗っけてみる予定。(何やるにしても個人の趣味だしなんでも良し)
